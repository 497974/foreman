02

部署到阿里云 FC + 录部署证明

证明「后端真的跑在阿里云上」——用免费额度即可
约 50 分钟
核心思路：FC 部署本身免费。部署好后在云上用「演示模式」跑一次就证明了后端在阿里云运行——不需要任何模型额度。「用了阿里云 API」这一项由代码文件链接单独满足（下方 03 步）。

确认部署包已就绪

在项目目录跑这条，看到 foreman-fc.zip 且大小约 4.9MB 即可（已自包含 openai 依赖，上传即用）：

python fc/build_package.py

登录阿里云国际版，切到新加坡

打开 alibabacloud.com 控制台，右上角地域选 Singapore (ap-southeast-1)（和 Foreman 调用的 DashScope 同区）。搜索 Function Compute / 函数计算，首次进去点「开通服务」（免费）。

创建函数：Web 函数 + 自定义运行时

创建函数 → 类型选 Web 函数（Web function） → 运行时选 自定义运行时（Custom Runtime），基础镜像挑一个 Python 3.10+ 的。

找不到一模一样的字？不同控制台版本文字略有出入，认概念：「Web 函数」+「自定义运行时」这对组合就对了。

上传代码 zip

代码上传方式选「上传 zip 包」，选 Foreman/fc/foreman-fc.zip。

设置入口和端口

启动命令 / 入口填 ./fc/bootstrap（若预填了 ./bootstrap 且识别不到，就改成前者）。端口填 9000。

端口必须是 9000：包里的 bootstrap 写死了 serve.py --host 0.0.0.0 --port 9000，控制台端口要和它一致。

环境变量 + 单实例

配置 → 环境变量，加一条 DASHSCOPE_BASE_URL = https://dashscope-intl.aliyuncs.com/compatible-mode/v1。有 Qwen key 就再加 DASHSCOPE_API_KEY（证明用演示模式，没有也行）。扩缩容里把最大实例数设为 1（保证 runs 目录会话一致）。

部署，拿公网地址

保存/部署 → 在「触发器」标签页找到形如 https://xxxx.fcapp.run 的公网地址。浏览器打开它，应该看到和本地一样的 Foreman 控制台。

录 30–60 秒「部署证明」录屏

这是独立于 demo 视频的一段短录屏。画面依次给到：

① FC 控制台的函数配置页（能看到 地域=新加坡、自定义运行时）→ ② 那个 fcapp.run 地址在浏览器地址栏里打开着 Foreman → ③ 勾演示模式跑一次，状态墙变绿。这就证明「后端在阿里云上真实运行且能端到端工作」。

两条红线：①录屏里绝不能出现 API Key（环境变量页别录进去/打码）。②录完立刻停掉或删除函数，别让它挂着产生费用。
费用守门：FC 免费额度覆盖一次演示。别开高并发/预留实例。录完就停。
