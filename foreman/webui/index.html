Foreman — your AI foreman

Give it a long requirements list. It plans the work, hires an AI to build each piece, REFUSES to accept anything that fails a real test, and keeps going until everything is green — even across crashes.

1Paste your listType requirements one per line in "New Run" — or pick a ready-made template.
2Press StartWatch the status wall: each cell is one task going grey → amber → green.
3Collect the goodsWhen all cells are green, download the finished code as a zip. Every item was proven by a real test.
Everything it can do — and how to use each one

📝 New Run

Turns a requirements list into a fully built, fully tested project.

HOW: left panel → paste your list (one item per line) → Start Foreman.

🎬 Demo mode

The whole product with fake workers — zero API key, zero cost, offline.

HOW: tick "Demo mode" before starting. Great first click.

📋 Templates

Ready-made requirement lists (expense API, World Cup predictor…).

HOW: "Template" dropdown → pick one → it fills the box for you.

🧠 Per-role models

Planner, builder and inspector can each use a different Qwen model.

HOW: three dropdowns under the checklist. Defaults are already set to free-quota models — just leave them.

📂 Edit a REAL project

Points Foreman at your actual code repo — safely: it works on its own branch, never touches yours, never pushes.

HOW: fill "Existing project folder" + tick the confirmation. Review with git diff afterwards; merging is always your call.

🧱 Status wall + live feed

One cell per task; the event feed shows every claim, verdict and retry as it happens.

HOW: click any run in the "Runs" list. Green=done, amber=working, red=stuck.

✅ Real test gates

A task counts as done ONLY after its tests actually pass. "Trust me, it's done" doesn't fly here.

HOW: automatic. Seeing REJECT in the feed is the system working — the fix comes in the next attempt.

⚖️ Appeals court

If the builder thinks a rejection was unfair, it can appeal once — a stronger model reads the evidence and rules.

HOW: automatic. Watch for amber DISPUTE / ARBITRATION badges in the event feed.

⏸️ Stop & Resume

Stop gracefully at the next task boundary. Crashes, closed laptops, dead quota — nothing loses progress.

HOW: Stop / Resume buttons on the run header. Finished tasks are never redone.

💰 Live cost meter

Each run shows tokens spent and an estimated dollar cost, live.

HOW: automatic, on the run header (≈$0.0123 · 45,678 tok). Estimate only — not a bill.

📦 Download & archive

Grab the finished code as a zip; archive old runs to keep the list tidy.

HOW: buttons on each run row / run header after it finishes.

🌐 Web search + Hermes

The builder can search the live internet (no key needed). Power users can even swap in the hermes-agent as the builder.

HOW: web search is automatic when a task needs facts. Hermes: see docs/HERMES.md (one env var).

Honest limits: real runs need a DashScope API key in the .env file (Demo mode doesn't). Tests are pytest-style today. If a task fails 3 times it stops and waits for you instead of looping forever — that's a feature.
